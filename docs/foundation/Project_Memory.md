# Project Memory

## 1. 목적
SnapTXT의 목적은 책을 Office Lens로 촬영하고 → PC SnapTXT에서 텍스트를 추출한 뒤 → 내가 만든 웹페이지에 업로드하여 브라우저(Chrome, Safari 등)에서 편하게 읽거나 오디오북처럼 들을 수 있게 만드는 것이다. 핵심은 **책 내용을 소비 가능한 형태로 만드는 것**이다.

## 2. OCR 전략
- 고정밀 AI OCR은 비용이 크고 다중 페이지 처리에 비효율적이다.
- SnapTXT는 “완벽한 OCR”보다 “실용적인 OCR”을 선택한다.
- 정확도가 약간 낮아도 저비용·다페이지 처리가 가능해야 한다.

## 3. 핵심 철학
- 목표는 완벽한 텍스트가 아니라 **읽을 수 있는 텍스트**다.
- 띄어쓰기 정리, 문단 구조 복원, 깨진 문자 수정 등 **후처리**가 설계의 중심이다.
- 후처리를 통해 읽기 자연성을 확보하면 사용자는 즉시 책을 소비할 수 있다.

## 4. 역할 분리
- SnapTXT: 텍스트를 추출하고 정리한다.
- 웹: 정리된 텍스트를 활용해 읽기/듣기/하이라이트/학습 경험을 제공한다.
- 학습 경험의 완성은 웹에서 이뤄진다.

## 5. 운영 원칙
- 비용 최소화
- 다중 페이지 처리 가능
- 읽기 자연성 중심
- 후처리 우선 설계

## 6. 실행 기준
- **시간**: 촬영 후 1분 내에 웹에서 읽기/듣기가 가능해야 한다.
- **비용**: 페이지당 추가 비용 없이 일일 수십 페이지를 처리할 수 있어야 한다.
- **품질**: 사용자 피드백 기준으로 “읽을 수 있다”는 평가를 90% 이상 유지한다.
- **문서**: 기능 변경 시 본 문서·Architecture·Current_Work를 즉시 갱신한다.

## 7. 결론
SnapTXT는 “완벽한 OCR 도구”가 아니라 **책을 읽고 들을 수 있게 만드는 실용적 텍스트 생성 도구**다. 이 방향을 잃지 않는 것이 프로젝트의 기준점이다.
